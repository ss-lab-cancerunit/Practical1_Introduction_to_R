History of R and Bioconductor

R is a programming language, and a development environment for data manipulation, statistical computing, and graphical display. R is an open source implementation that evolved from the S and S-PLUS languages developed at Bell laboratories by R. Becker, J. Chambers and A. Wilks. It was created by Robert Gentleman & Ross Ihaka as a programming language to teach introductory statistics at the University of Auckland.

Robert Gentleman

Ross Ihaka

Bioconductor https://www.bioconductor.org/ on the other hand is both an open source and open development project and a package repository for R based software packages for genomic and biological data analysis. There are two releases of bioconductor annually (and a development version that is less stable and tied to the developmental version of R). Other R package repositories (resources where R based packages can be downloaded from) include CRAN https://cran.r-project.org/ and R-Forge https://r-forge.r-project.org/

R objects and Assignment

In R, all different kinds of data are stored in containers called R-objects.

# assign this calculation to a variable called values. <- is the assignment operator
sin(pi/2)


# You can pick any name that conforms to R object naming rules
# Object names must start with a letter and can only contain letters, numbers, _, and .. 

value <- sin(pi/2)

#value variable stores the value of the calculation. Use sensible names for variables.

print(value)

R object names:

  • Object names are case sensitive: object firstPrimes is not the same as firstprimes or Firstprimes

  • They can contains both numbers or letters but should begin with a letter, e.g. 6firstprimes not a valid object name.

  • You can use underscores. Other special characters should be avoided.

  • Avoid using function names as object names.

  • snake_case can be used

FUNCTIONS

One of the best ways to improve your programming skills is to write functions. Functions allow you to automate common tasks in a more powerful and general way than writing single lines of code. Writing a function has four big advantages over using copy-and-paste:

  1. You can give a function an evocative name that makes your code easier to understand.

  2. As requirements change, you only need to update code in one place, instead of many.

  3. You eliminate the chance of making incidental mistakes when you copy and paste (i.e. updating a variable name in one place, but not in another).

Information can be passed into functions as arguments. To call a function, use the function name followed by parenthesis, like my_function():

my_function <- function(fname) {
  paste("Homo", fname)
  
}

my_function("sapiens")
my_function("denisova")
my_function("neanderthalensis")
